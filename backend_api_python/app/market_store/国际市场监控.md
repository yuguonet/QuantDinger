# Market Feather Store
这是https://github.com/brokermr810/QuantDinger的一个插件,文件位置在back_api_python/app/market_store/
作用是对全球市场一个整体分类评估和异动报警功能

使用的是QuantDinger的财经数据源
基于 Feather 格式的本地行情数据持久化模块，可独立运行采集器，也可被其他代码 import 使用。
https://github.com/yuguonet/QuantDinger/tree/main/backend_api_python/app/market_store

## 文件结构

```
market_feather_store/
├── .env.example          # 环境变量配置
├── market_store.py       # 核心模块（采集器 + 存储 + 查询 + 异常检测）
├── example_analysis.py   # 使用示例
└── data/feather/         # 数据文件（自动创建）
    ├── market_2026-04-17.feather
    ├── market_2026-04-18.feather
    └── ...
```

## 快速开始

```bash
# 1. 安装依赖
pip install pandas pyarrow yfinance requests

# 2. 复制配置
cp .env.example .env
# 编辑 .env 设置采集间隔和保留天数

# 3. 单次采集测试
python market_store.py fetch

# 4. 查看统计
python market_store.py stats

# 5. 启动定时采集（后台运行）
python market_store.py
# Ctrl+C 停止
```

## CLI 命令

| 命令 | 说明 |
|------|------|
| `python market_store.py` | 启动定时采集循环 |
| `python market_store.py fetch` | 单次采集并保存 |
| `python market_store.py stats` | 查看存储统计 |
| `python market_store.py query --hours 24 --category crypto` | 查询数据 |

## 作为库使用

```python
from market_store import MarketStore

store = MarketStore()

# 最近 24 小时所有数据
df = store.query(hours=24)

# 只查加密货币
df = store.query(category="crypto", hours=24)

# 日期范围
df = store.query(start="2026-04-10", end="2026-04-18")

# 按标的查
df = store.query(category="crypto", symbol="BTC")

# 情绪指标
df = store.query(category="sentiment", hours=48)

# 直接用 pandas 分析
df.groupby("symbol")["price"].last()
```

## 数据类别 (category)

| 类别 | 说明 | 标的示例 |
|------|------|---------|
| `indices` | 全球指数 | ^GSPC(标普), ^DJI(道琼斯), ^N225(日经) |
| `crypto` | 加密货币 | BTC, ETH, SOL, BNB |
| `forex` | 外汇 | EUR/USD, USD/JPY, USD/CNH |
| `commodities` | 大宗商品 | GC(黄金), CL(原油), SI(白银) |
| `sentiment` | 情绪指标 | fear_greed(恐贪), VIX, DXY |

## 自动特性

- **按天分区**: 每天一个文件 `market_YYYY-MM-DD.feather`
- **损坏自愈**: 文件读取失败自动删除重建
- **去重**: 同一 timestamp + category + symbol 自动去重
- **自动清理**: 超过保留天数的文件自动删除
- **原子写入**: 先写 .tmp 再 rename，防止写入中断导致损坏
- **并发采集**: 5 个数据源并行拉取
- **急剧变化检测**: 每次写入时与历史前 N 条比对，异常时输出告警

## 急剧变化检测

每次写入新数据时，自动对每个标的与历史最近 15 条（可配）数据比对：

**检测算法**:
1. 取历史最近 N 条的 price 变动率序列，算均值 μ 和标准差 σ
2. 当前变动率 = (新价 - 上次价) / 上次价 × 100
3. 若 |变动率| > 最小阈值 **且** |变动率 - μ| > z_score × σ → 告警

**告警输出示例**:
```
🟡 🔺暴涨 | [crypto] BTC (Bitcoin) | 65000.00 → 70500.00 | 变动 +8.462% | z=3.2 (μ=0.12 σ=1.85)
🔴 🔻暴跌 | [indices] ^GSPC (标普500) | 5200.00 → 4950.00 | 变动 -4.808% | z=4.1 (μ=0.05 σ=1.20)
```

**配置项** (.env):

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `FEATHER_ANOMALY_WINDOW` | 15 | 回看最近 N 条历史 |
| `FEATHER_ANOMALY_ZSCORE` | 2.5 | z-score 阈值（越大越严格） |
| `FEATHER_ANOMALY_MIN_PCT` | 2.0 | 最小变动%，低于此值不告警 |
| `FEATHER_ANOMALY_COOLDOWN` | 600 | 同标的告警冷却秒数（防刷屏） |

**作为库使用检测结果**:
```python
from market_store import MarketStore, fetch_all

store = MarketStore()
df = fetch_all()

# 手动检测（不写入）
alerts = store.detect_anomalies(df)
for a in alerts:
    print(a["message"])

# 正常写入时自动检测并 log
store.append(df)
```

## 容错机制

`append()` 内置完整的容错流水线，防止因数据采集问题导致误报警：

```
fetch_all() → 完整性检查 → 合理性校验 → 跳变校验 → 缺失回填 → 写入 → 异常检测
```

### 1. 采集完整性检查

每次采集后统计实际获取数 vs 预期数：

```
预期: indices=10, crypto=12, forex=8, commodities=6, sentiment=3
实际: indices=10, crypto=8,  forex=8, commodities=6, sentiment=2
完整率: 87.2% (34/39)
```

- 完整率 ≥ `FEATHER_MIN_FETCH_RATIO`（默认 50%）→ 正常继续
- 完整率 < 阈值 → 告警但**不触发异常检测**，防止因数据不全导致误判

### 2. 价格合理性校验

过滤明显错误的数据：
- `price <= 0` → 丢弃
- `price` 不在合理范围内（如 BTC 报价 0.01 或 1亿）→ 丢弃
- `price` 为 NaN / inf → 丢弃

### 3. 单次跳变校验

新价格 vs 上次价格，变动超过 `FEATHER_SANITY_MAX_PCT`（默认 50%）视为采集错误：

```
sanity reject: crypto BTC jump 98.5% (65000.00 → 129025.00) exceeds 50% cap — likely bad data, discarding
```

### 4. 缺失数据回填

采集不完整的标的，自动沿用上次数据补全：

```
本次采集到 crypto: 8/12
fill_missing: backfilled 4 symbols with last known values (ETH, SOL, DOGE, AVAX)
```

回填数据标记 `_filled=True`，不参与异常检测。

### 5. 异常检测前置条件

异常检测**仅在以下条件全部满足时**才运行：
- ✅ 完整率 ≥ `FEATHER_MIN_FETCH_RATIO`
- ✅ 至少有 1 行通过合理性校验的真实数据
- ❌ 完整率不足 → 跳过检测，日志提示 `anomaly detection SKIPPED`

### 配置项

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `FEATHER_SANITY_MAX_PCT` | 50.0 | 单次最大合理变动% |
| `FEATHER_MIN_FETCH_RATIO` | 0.5 | 最低采集完整率 |
| `FEATHER_FILL_MISSING` | 1 | 缺失回填开关 |
