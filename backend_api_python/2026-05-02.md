# 2026-05-02 记录

## A股数据存储方案（已确定）

### 数据库架构
- 同一台 PG 实例，两个库
- **market_db**：A股全量数据（只读为主，每日增量写入）
- **strategy_db**：用户自己的策略/信号/回测结果/持仓（已有结构）
- 通过 **postgres_fdw** 桥接，strategy_db 映射 market_db 的表

### market_db 核心设计
- **唯一数据源**：15分钟K线，按年分区（5个分区）
- **日线不单独存储**：通过 VIEW 从15分钟聚合
- 数据规模：5年全量约 6-7GB（CSV原始约2.5GB）
- 增量方式：UPSERT（ON CONFLICT DO UPDATE）
